快速开始

几分钟内完成接入

1

获取 API 密钥

注册账号并在控制台创建您的 API 密钥

2

安装 SDK 或使用 cURL

使用任意 OpenAI 兼容 SDK 或直接发起 HTTP 请求

3

发起首次请求

向我们的统一接口发送一个对话补全请求

身份认证

所有 API 请求都需要使用 Bearer Token 进行身份认证

Authorization: Bearer hiai-your-api-key

请妥善保管您的 API 密钥，切勿在客户端代码中暴露。

API 参考

POST/v1/chat/completions

对话补全

使用指定模型创建对话补全

Parameters: model（必填）、messages（必填）、temperature、top_p、max_tokens、stream

GET/v1/models

模型列表

列出所有可用模型

Terminal
curl https://api.wuguhub.com/v1/chat/completions \
  -H "Authorization: Bearer hiai-xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "user", "content": "Hello!"}
    ]
  }'

速率限制

速率限制因套餐而异，按 API 密钥独立计算

请查看响应头中的 X-RateLimit-Limit、X-RateLimit-Remaining 和 X-RateLimit-Reset

See Pricing for rate limit details by plan.

错误码说明

标准错误响应遵循 OpenAI 错误格式

Status CodeDescription
400请求错误 — 参数无效
401未授权 — API 密钥无效或缺失
404未找到 — 模型或资源不存在
429请求过多 — 已达速率限制
500服务器错误 — 内部错误
502上游错误 — LLM 提供商返回错误
